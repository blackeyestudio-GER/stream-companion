    How to use in OBS

    index.html is only the editor. Each config folder contains its own output.html, and that is the file you should add to OBS.

    The output file reads its local config.json automatically. Dynamic speech still comes from #say=… appended to the OBS URL by Streamer.bot.

    Settings in the editor are saved automatically to the currently opened config, including default if that is the config you chose to edit.

    Recommended workflow

    Use output.html in OBS and let Streamer.bot update only the hash fragment.

    file:///.../configs/default/output.html#say=Hey%20everyone%20thanks%20for%20watching

    Streamer.bot C# example

    Paste this into a C# action, then replace the scene name, browser source name, and local file path.

    public bool Execute()
{
    string scene = "Your Scene";
    string source = "Your Browser Source";
    string baseUrl = "file:///C:/stream-companion/configs/default/output.html";

    CPH.TryGetArg("rawInput", out string speechText);
    speechText ??= "";

    string url = baseUrl + "#say=" + CPH.UrlEncode(speechText);
    CPH.ObsSetBrowserSource(scene, source, url);
    return true;
}

    Use the message text you want to show as rawInput or swap in another variable. Spaces and special characters are handled by CPH.UrlEncode(...).

    Chat trigger variant

    Use this when the action is triggered directly from Twitch or YouTube chat messages. Those triggers provide the chat text as %message%.

    public bool Execute()
{
    string scene = "Your Scene";
    string source = "Your Browser Source";
    string baseUrl = "file:///C:/stream-companion/configs/default/output.html";

    CPH.TryGetArg("message", out string speechText);
    speechText ??= "";

    string url = baseUrl + "#say=" + CPH.UrlEncode(speechText);
    CPH.ObsSetBrowserSource(scene, source, url);
    return true;
}

    For command-based actions, use rawInput instead. For direct Twitch Chat Message and YouTube Chat Message triggers, use message.
